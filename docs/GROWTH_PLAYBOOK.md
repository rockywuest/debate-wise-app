# Growth Playbook

Last updated: April 3, 2026

This playbook focuses on discoverability, contributor conversion, and distribution momentum.

## 1) Discoverability Baseline

- Keep repository description and topics precise and keyword-rich.
- Maintain a concise `llms.txt` for machine indexing.
- Keep README top section scannable within 30 seconds.
- Maintain social preview image in GitHub settings.

## 2) Contributor Conversion

- Keep three pinned `good first issue` tickets always open.
- Respond to first-time contributor PRs within 72 hours.
- Keep CI green on `main` to reduce contributor friction.

## 3) Distribution Channels

- Post monthly roadmap/release update issue and share it publicly.
- Share release highlights in:
  - GitHub discussions
  - LinkedIn/X short thread format
  - relevant civic-tech and open-source communities
- Cross-link to concrete starter issues in every announcement.

## 4) Viral Loop Design

- Every release note includes:
  - what improved for users
  - one screenshot/GIF
  - one direct "how to contribute this week" CTA
- Every roadmap post includes:
  - measurable milestone from prior month
  - one high-impact community ask

## 5) Metrics to Watch

- Stars per month
- New contributors per month
- PR median time to first maintainer response
- PR merge throughput
- Issue close rate
